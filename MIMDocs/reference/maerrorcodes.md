# <a name="management-agent-run-error-codes"></a>Códigos de erro de execução do agente de gerenciamento

As tabelas a seguir contêm códigos de erro que podem aparecer na interface do usuário do Synchronization Service Manager no MIM (Microsoft Identity Manager) 2016, bem como descrições de cada um desses erros.

## <a name="connection-errors"></a>Erros de conexão

| Erro do                  | Descrição                                    |
|--------------------------|--------------------------------------------|
| failed=connection         | A conexão com o diretório conectado falhou por um motivo que não é a autenticação. Por exemplo, a rede não está disponível ou o servidor de destino está offline.                                                                                                                                                                                                                                                                                                                      |
| dropped-connection        | A conexão entre o agente de gerenciamento e o diretório conectado não existe mais. O agente de gerenciamento tentará se reconectar ao diretório conectado em várias instâncias.                                                                                                                                                                                                                                                                                                         |
| failed-authentication     | A autenticação não pode ser realizada usando as credenciais fornecidas.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| failed-permission         | Direitos insuficientes para acessar um contêiner no diretório conectado. Esse erro é esperado apenas para agentes de gerenciamento LDAP (Lightweight Directory Access Protocol) que pesquisam por contêineres diferentes de diretório conectado.                                                                                                                                                                                                                                                              |
| failed-search            | Uma pesquisa de tabela ou contêiner falhou com um erro inesperado.                                                                                                                                                                                                                                                                                                                                                                                                                            |
| warning-no-watermark      | O agente de gerenciamento não consegue ler a marca d'água ao realizar uma importação completa. Esse erro será esperado somente no agente de gerenciamento para Sun ONE Directory Server 5.1 (conhecido antes como iPlanet Directory Server) quando a configuração inicial do agente de gerenciamento for concluída, e o log de alterações do diretório conectado estiver habilitado. Depois, quando o log de alterações do diretório conectado for desativado, se a configuração do agente de gerenciamento não for atualizada, esse aviso ocorrerá quando uma importação completa for realizada. |
| no-start-partition-delete | A partição LDAP configurada originalmente para este agente de gerenciamento não existe mais. Esse erro retorna quando a partição tiver sido excluída, ou se a partição for excluída e recriada com o mesmo nome. No último caso, mesmo que o nome seja o mesmo, o GUID da partição de erro terá sido alterado.                                                                                                                                                                           |

## <a name="discovery-errors"></a>Erros de descoberta

| Erro do                   | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|-------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| missing-change-type      | Esse erro retorna durante a execução de uma importação delta por agentes de gerenciamento de banco de dados e com base em arquivo, e também pelo agente de gerenciamento para servidores de diretório Sun e Netscape, quando o valor de coluna do tipo de alteração (adicionar, modificar, excluir) não estiver presente.                                                                                                                                                                                                                                          |
| invalid-change-type      | Esse erro retorna durante a execução de uma importação delta por agentes de gerenciamento de banco de dados e com base em arquivo, e também pelo agente de gerenciamento para servidores de diretório Sun e Netscape, quando o valor de coluna do tipo de alteração não corresponder com a lista de tipos de alteração válidos. Também retorna de uma importação completa LDIF (Formato de Troca de Dados LDAP) quando houver um campo de tipo de alteração com um valor diferente de adicionar.                                                                                  |
| multi-valued-change-type | Esse erro retorna durante a execução de uma importação delta por agentes de gerenciamento de servidores de diretório Sun e Netscape, quando mais de um valor para o tipo de alteração estiver presente.                                                                                                                                                                                                                                                                                                        |
| need-full-object        | Esse erro retorna durante a execução de uma importação delta de um agente de gerenciamento baseado em arquivo ou ao retomar de um agente de gerenciamento baseado em arquivo. Ele indica que o agente de gerenciamento enviou uma modificação em um objeto que não pode ser localizado no espaço do conector. O mecanismo de sincronização está solicitando os valores atuais de todos os atributos no objeto. Como esta é uma importação de um arquivo, essas informações não estão disponíveis. Uma importação completa deve resolver esse problema. |
| missing-dn              | Esse erro retorna para agentes de gerenciamento baseado em arquivo (ou seja, agentes de gerenciamento para LDIF, DSML ou arquivos simples com atributos de nome de domínio configurados) quando não há valor de nome de domínio. Ele também retorna no caso de um log de alterações corrompido do Sun ONE Directory Server no qual o atributo de nome de domínio está ausente. Ele indica que o agente de gerenciamento pode ler o elemento e analisá-lo, mas que não há um valor de nome de domínio para o objeto.                           |
| dn-not-ldap-conformant   | Esse erro retorna quando um agente de gerenciamento para LDAP, LDIF, DSML ou arquivo simples com um atributo de nome de domínio configurado relata um valor de nome de domínio que não se adequa à especificação de LDAP.                                                                                                                                                                                                                                                                                |
| invalid-dn              | Esse erro retorna quando um agente de gerenciamento informa que um nome de domínio não atende a uma restrição FIM, que inclui:   <br/>&#176; Um ou mais caracteres que não têm permissão do FIM <br/>&#176; Um nome relativo distinto (também conhecido como RDN) vazio <br/>&#176; Um nome distinto relativo que excede o máximo para o FIM <br/>&#176; O número de níveis de hierarquia do nome de domínio excede o máximo para o FIM                                                                                                                                                                                                                                                                                                                                                            |
| missing-anchor-component | Esse erro é retornado por agentes de gerenciamento de banco de dados e baseado em arquivo, e também pelo agente de gerenciamento de servidores de diretório Sun e Netscape, quando a âncora não pode ser construída devido à falta de valores em um ou mais atributos de regra de construção de âncora.   |
| multi-valued-anchor-component | Esse erro é retornado pelo agente de gerenciamento para servidores de diretório Sun e Netscape se eles não conseguirem construir a âncora porque um atributo de regra de construção de âncora tem mais de um valor. |   
|anchor-too-long | Esse erro é retornado por agentes de gerenciamento de banco de dados e baseado em arquivo, e também pelo agente de gerenciamento de servidores de diretório Sun e Netscape, quando a construção de âncora produzir uma âncora que ultrapassa o limite de tamanho máximo para o FIM.|
|duplicate-object | Esse erro é retornado em importações completas por agentes de gerenciamento de banco de dados e com base em arquivo quando um objeto com a mesma âncora já tiver sido reportado para o mecanismo de sincronização durante essa execução.|

>[!Note]
A obsolescência de objetos de espaço do conector só ocorrerá se a etapa de execução atual tiver sido concluída com Sucesso, Concluída com falha de sincronização, Concluída com aviso ou Concluída com transitório.

|Erro do| Descrição |
|----|----|  
|missing-object-class | Esse erro é retornado por um agente de gerenciamento baseado em arquivo (ou seja, um agente de gerenciamento para DSML, LDIF ou um arquivo simples com um atributo de classe de objeto configurado), ou para o agente de gerenciamento de servidores de diretório Sun e Netscape, se houver um log de alterações corrompido. Isso indica que o agente de gerenciamento não pode ler um valor do atributo de classe de objeto. |
|missing-object-type | Esse erro retorna ao retomar a importação de um arquivo descartado corrompido. Esse erro não deve ser encontrado durante a operação normal.        |
|unmappable-object-type  | Esse erro é retornado por um agente de gerenciamento baseado em arquivo quando ele lê um objeto que tem um conjunto de valores de classe de objeto sem correspondência possível com qualquer um dos mapeamentos de prefixo. |
|parse-error | Esse erro é retornado pelo agente de gerenciamento de servidores de diretório Sun e Netscape no modo de delta, e por agentes de gerenciamento baseado em arquivo, quando eles não conseguem analisar uma entrada. O elemento `<entry-number>` (na maioria dos casos `<line-number>` e `<column-number>`) estará presente para ajudar a localizar o erro. O elemento `<attribute-name>` pode estar presente. O agente de gerenciamento de servidores de diretório Sun e Netscape finaliza a execução quando encontra esse elemento. Os agentes de gerenciamento baseado em arquivo registram o erro de descoberta e continuam. |   
|read-error | Esse erro é retornado por agentes de gerenciamento baseado em chamada quando há um erro genérico de leitura de um objeto específico. Isso geralmente causa o encerramento da execução. O elemento de erro da fonte de dados conectada está presente, e você pode usá-lo para solucionar o problema.    |
|staging-error | Esse erro é retornado por grande parte dos agentes de gerenciamento. Indica que o mecanismo de sincronização não conseguiu preparar o delta no espaço do conector. O servidor cria um log de eventos que fornece informações sobre o problema e que pode ser usado para solucionar problemas. A maioria dos agentes de gerenciamento continua a execução da importação quando o erro é registrado, mas o agente de gerenciamento para execuções delta de Sun e Netscape para, pois falhas no processamento do log de alteração podem causar um estado inconsistente no espaço do conector. Esse erro não deve ser encontrado durante a operação normal.  |
|invalid-modification-type | Esse erro retorna durante a importação delta em um agente de gerenciamento LDIF quando um tipo de modificação de nível do objeto não é um dos tipos de modificação LDIF padrão, ou quando há um tipo de modificação nonreplace na objectclass, como add: objectclass ou delete: objectclass.    |
|conflicting-modification-types        | Esse erro é retornado pelo agente de gerenciamento LDIF e indica que tipos de modificação no nível de atributo diferentes foram encontrados no mesmo registro (neste caso, o nome do atributo que produziu os tipos conflitantes será relatado) ou que vários deltas LDIF de substituição foram vistos no mesmo arquivo, por exemplo:    <br/> [Exemplo de arquivo LDIF](./media/maerrorcodes/conflict-modification-types.jpg) |
|multi-single-mismatch | Esse erro é retornado por um agente de gerenciamento baseado em arquivo quando ele reporta mais de um valor add, ou mais de um valor delete para um atributo que está definido no FIM como um atributo de valor único. Esse erro pode indicar que o esquema de fonte de dados conectado, e que está armazenado no FIM, está especificado incorretamente (agentes de gerenciamento baseado em arquivo) ou está desatualizado com relação ao esquema atual. Inclui um elemento `<attribute-name>` que fornece o contexto do erro.    |
|invalid-attribute-value | Esse erro é retornado por um agente de gerenciamento baseado em chamada quando ocorre a leitura do valor de um atributo que não está de acordo com o tipo de atributo declarado no esquema. Inclui um elemento `<attribute-name>` que fornece o contexto do erro.  |
|invalid-base64-value| Esse erro é retornado pelos agentes de gerenciamento para LDIF, DSML e servidores de diretório Sun e Netscape quando encontram uma cadeia de caracteres base64 inválida.     |
|invalid-numeric-value | Esse erro é retornado por agentes de gerenciamento baseado em arquivo e pelo agente de gerenciamento para LDAP quando eles não conseguem analisar um valor numérico. Inclui um elemento `<attribute-name>` que fornece o contexto do erro. |
|invalid-boolean-value | Esse erro é retornado por agentes de gerenciamento baseado em arquivo e pelo agente de gerenciamento para LDAP quando eles não conseguem analisar um valor booliano. Inclui um elemento `<attribute-name>` que fornece o contexto do erro.  |
|reference-value-not-ldap-conformant | Esse erro é retornado pelos agentes de gerenciamento para LDAP, LDIF e DSML ou arquivo simples (com um atributo de nome de domínio configurado) quando um valor de nome de domínio não atende à especificação de LDAP. Essa mensagem de erro inclui um elemento `<attribute-name>` que fornece o contexto do erro.|
|invalid-reference-value | Esse erro é retornado por um agente de gerenciamento quando um nome de domínio não atende às restrições de FIM, que incluem:  <br/>&#176; Um ou mais caracteres que não têm permissão do FIM   <br/>&#176; Um nome relativo distinto (também conhecido como RDN) vazio  <br/>&#176; Um nome distinto relativo que excede o máximo para o FIM  <br/>&#176; O número de níveis de hierarquia do nome de domínio excede o máximo para o FIM                |
|unsupported-value-type | Esse erro é retornado pelo agente de gerenciamento DSML ou LDIF quando o tipo do valor fornecido no arquivo é incompatível com o tipo do atributo, incluindo:    <br/>&#176; Um valor de URI ou URL é fornecido para um atributo que não é de cadeia de caracteres, ou para qualquer palavra-chave reservada como dn, objectclass ou changetype.  <br/>&#176; Um valor base64 é fornecido para o atributo changetype.  <br/>&#176; Um valor de cadeia de caracteres que contém caracteres que não são ASCII é fornecido para um atributo binário. |

## <a name="synchronization-errors"></a>Erros de sincronização

| Erro do               |Descrição      |
|---------|-------------|
| extension-dll-exception  | Esse erro ocorrerá se uma extensão de regra gerar uma exceção. Se você encontrar esse erro, observe o elemento \<exception-error-info\> para examinar a pilha de chamadas da exceção. Em alguns casos, o `<rule-error-info>` está presente e fornece mais informações sobre qual regra estava sendo processada quando o erro ocorreu.        |
| extension-dll-crash | Esse erro ocorre quando o processo que está executando a extensão de regra é encerrado inesperadamente. Esse erro só pode ocorrer quando uma extensão de regra está sendo executada fora do processo. Uma possível causa para o valor desse erro é que a extensão de regras está chamando um código que causa uma violação de acesso. |
| extension-dll-timeout                          | Esse erro ocorre se o cliente tiver configurado um tempo limite para a extensão, e a chamada em um único ponto de entrada do código de extensão do cliente excede o limite configurado. O `<exception-error-info>` fornecerá informações contextuais sobre o motivo de o ponto de entrada estar sendo chamado quando atingiu o tempo limite. Em alguns casos, o <rule-error-info> estará presente e fornecerá mais informações sobre qual regra estava sendo processada quando o erro ocorreu. Observe que, quando você está depurando o processo que está executando a extensão, os tempos limite não são impostos.  |              
| extension-projection-object-type-not-set        | Esse erro ocorre quando a implementação do método **IMASynchronization.ShouldProjectToMV** na extensão de regras não especifica o tipo de objeto do metaverso.                    |
| extension-projection-invalid-object-type        | Esse erro ocorre quando a implementação do método **IMASynchronization.ShouldProjectToMV** na extensão de regras define o valor do tipo de objeto de metaverso de saída como um valor que não está listado no Designer de Metaverso do Synchronization Service Manager. Verifique se o método usa um dos valores de tipo de objeto especificado.   |
| extension-join-resolution-invalid-object-type   | Esse erro ocorre quando a implementação do método **IMASynchronization.ResolveJoinSearch** na extensão de regras define o valor do tipo de objeto de metaverso de saída como um valor que não está listado no Designer de Metaverso do Synchronization Service Manager. Verifique se o método define o valor do tipo de objeto de metaverso de saída como um dos valores de tipo de objeto listados.   |
| extension-join-resolution-index-out-of-bounds   | Esse erro ocorre quando uma implementação do método **IMASynchronization.ResolveJoinSearch** na extensão de regras define um valor de índice que é negativo, ou maior ou igual ao, número de objetos do metaverso.                     |
| extension-provisioning-call-limit-reached       | Esse erro ocorre quando o método **IMASynchronization.Provision** é chamado mais de 10 vezes durante a sincronização de um único objeto. Esse método pode ser chamado mais de uma vez se a lógica do cliente no método Provision desprovisionar um objeto, e se houver um cancelamento de atributo que causa uma alteração no objeto do metaverso, resultando em uma nova chamada para Provision. O limite de 10 chamadas para o método Provision é configurado parar interromper possíveis notas de provisionamento infinitas.                                                                                                                                               |
| extension-deprovisioning-invalid-result         | Esse erro ocorre quando uma implementação do método **IMASynchronization.Deprovision** retorna um valor de enumeração DeprovisionAction inválido. Verifique se o método retorna um valor válido.        |
| extension-entry-point-not-implemented           | Esse erro ocorre quando uma extensão de regras lança uma exceção **EntryPointNotImplementedException**. |
| extension-unexpected-attribute-value            | Esse erro ocorre quando uma extensão de regras lança uma exceção **UnexpectedDataException**.            |
| flow-multi-values-to-single-value              | Esse erro ocorre quando uma regra de fluxo de atributo de importação ou exportação configurada no Synchronization Service Manager tenta encaminhar um atributo com vários valores para um atributo de valor único. Esse erro retorna somente para regras de fluxo direto configuradas no Synchronization Service Manager. Se a regra de fluxo usar uma extensão de regras que encaminha vários valores para um atributo de valor único, a exceção **TooManyValuesException** será lançada.          |
| cs-attribute-type-mismatch                     | Esse erro ocorre quando o tipo do atributo importado não corresponde ao tipo de atributo especificado no esquema de agente de gerenciamento. Uma causa desse erro pode ser a desatualização do esquema armazenado de fonte de dados conectada com relação ao esquema real da fonte de dados conectada. Para atualizar o esquema armazenado de fonte de dados conectada, use o Synchronization Service Manager.|
| join-object-id-must-be-single-valued            | Esse erro ocorre quando o valor de atributo da fonte de dados usado para unir um objeto de metaverso por meio de uma regra de associação especificada nas propriedades de um agente de gerenciamento no Synchronization Service Manager contém mais de um valor. O valor de atributo da fonte de dados usado na regra de associação pode conter apenas um valor.     |
| dn-index-out-of-bounds       | Esse erro ocorre quando o valor de índice do componente de nome distinto usado em um fluxo de atributo de importação configurado nas propriedades de um agente de gerenciamento no Synchronization Service Manager for maior do que o número de componentes no nome distinto do objeto de origem.      |
| connector-filter-rule-violation                | Esse erro ocorre quando você executa uma operação de provisionamento de adição ou renomeação ou exporta um fluxo de atributo, e quando um objeto do conector se torna um objeto do desconector filtrado como resultado de uma configuração de filtro do conector. Esse valor não ocorre em objetos explícitos do conector.    |
| unsupported-container-delete   | O agente de gerenciamento está tentando excluir um objeto de contêiner durante o desprovisionamento. Os agentes de gerenciamento de FIM não conseguem excluir objetos de contêiner com objetos filho.            |
| ambiguous-import-flow-from-multiple-connectors   | Esse erro ocorre quando você tem vários conectores sob o agente de gerenciamento de origem conectado ao objeto de metaverso e uma regra declarativa de fluxo de atributo de importação é definida. Para importar os atributos usando um agente de gerenciamento com vários conectores para um objeto de metaverso, use uma extensão de regras para definir as regras de fluxo, em vez de configurar uma regra direta nas propriedades de um agente de gerenciamento.                                                                                                                                                                                                          |
| ambiguous-export-flow-to-single-valued-attribute | Esse erro ocorre quando a regra de fluxo de exportação, configurada nas propriedades de um agente de gerenciamento no Synchronization Service Manager, tenta encaminhar vários valores de um objeto de metaverso para um atributo de valor único.       |
| cannot-parse-object-id | O valor de cadeia de caracteres usado para pesquisar um objeto de metaverso em uma regra de associação especificada nas propriedades de um agente de gerenciamento no Synchronization Service Manager não está no formato GUID (identificador global exclusivo) correto. O formato GUID é {nnnnnnnn-nnnn-nnnnnnnn-nnnnnnnnnnnn}, em que n é um número hexadecimal. |
| unexported-container-rename  | A implementação do método IMVSynchronization.Provision ou IMASynchronization.Deprovision está tentando renomear um objeto de contêiner com um ou mais objetos filho não exportados.   |
| mv-constraint-violation           | Esse erro acontece durante o fluxo de um atributo de importação direta e o valor do atributo do espaço do conector excede as restrições de tamanho do atributo de metaverso. |
| locking-error-needs-retry   | Vários agentes de gerenciamento estão tentando sincronizar o mesmo objeto de espaço do conector. Execute novamente o agente de gerenciamento.     |
| unique-index-violation              | Um usuário está definindo manualmente um índice exclusivo em um atributo em uma tabela do metaverso. Não configure manualmente as tabelas do metaverso.    |
| encryption-key-lost                            | Os conjuntos de chave de criptografia não estão presentes no servidor que está executando o FIM.  |
| unexpected-error                              | Esse erro ocorre quando o mecanismo de sincronização tenta aplicar uma alteração no metaverso (incluindo o fluxo de atributo de provisionamento e exportação). Esse erro pode ocorrer somente durante execuções que aplicam alterações no metaverso. Verifique o log de eventos para obter mais informações.  |
| exported-change-not-reimported                 | Esse erro ocorre quando as alterações exportadas para um agente de gerenciamento não são reconfirmadas durante a execução de importação desse agente de gerenciamento. Um usuário ou processo de sistema que opera fora do FIM alterou os dados na fonte de dados conectada de uma forma que indica um problema de configuração, no qual a regra de fluxo de atributo de exportação está tentando encaminhar um valor para um objeto de fonte de dados conectada, mas a fonte de dados conectada redefine automaticamente o valor para algo diferente sem relatar um erro para o agente de gerenciamento. O elemento \<change-not-reimported\> indica quais alterações não foram reconfirmadas. |
| cannot-parse-dn-component                      | Esse erro é retornado por qualquer agente de gerenciamento que tenha um nome distinto ao estilo LDAP (também conhecido como DN) configurado, e a sincronização do espaço do conector para o metaverso falhou. Um componente de nome distinto não pode ser analisado por um mapeamento de dncomponent, pois ele não está no formato correto para o tipo de atributo de destino.|
| missing-partition-for-run-step                 | Esse erro indica que a partição especificada no perfil de execução não pode ser encontrada. Verifique se a partição não foi excluída ou renomeada.  |

## <a name="export-errors"></a>Erros de exportação

| Erro do                    |Descrição     |
|--------------------------|------------|
| cd-missing-object         | Esse erro retorna quando uma modificação de um objeto é exportada para a fonte de dados conectada, mas o objeto não pode ser encontrado na fonte de dados conectada. Ele retorna somente para agentes de gerenciamento baseado em chamada. Esse erro é causado por uma pessoa ou processo externo que excluiu um objeto da fonte de dados conectada fora do FIM.  |
| cd-existing-object        | Esse erro retorna quando uma adição é exportada para a fonte de dados conectada, mas o objeto já está presente na fonte de dados conectada. Ele retorna somente para agentes de gerenciamento baseado em chamadas e para agentes de gerenciamento de banco de dados relacional. |
| duplicate-anchor          | Esse erro retorna se a âncora de um objeto recentemente provisionado não for exclusiva. Ele retorna somente para agentes de gerenciamento de banco de dados e baseado em chamadas, bem como para o agente de gerenciamento de servidores de diretório Sun e Netscape. Se esse erro for encontrado, verifique as regras de construção de âncora para garantir que um valor exclusivo de âncora tenha sido definido para cada objeto.      |
| ambiguous-update          | Esse erro retorna quando o agente de gerenciamento não pode aplicar uma atualização ou exclusão delta porque a âncora não é exclusiva. Ele retorna somente para os agentes de gerenciamento do Microsoft SQL Server e do Banco de Dados Oracle. Se esse erro for encontrado, verifique as regras de construção de âncora para garantir que um valor exclusivo de âncora tenha sido definido para cada objeto. |

| password-set-disallowed   | Esse erro é retornado pelo agente de gerenciamento do ADAM (Modo de Aplicativo do Active Directory) quando a criptografia de senha estiver definida como sem criptografia ou como SSL (Secure Sockets Layer) de 128 bits, e o administrador não tiver feito uma substituição explícita para permitir a definição de senhas nesse cenário.      |
